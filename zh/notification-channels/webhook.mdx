简介：Webhook 是开发者友好的通知方式，支持将监控消息推送到您自定义的 HTTP 端点，实现与您的应用程序或工作流的深度集成。

## Webhook 消息格式

AlertDog 会以 POST 方法向您的 Webhook URL 发送 JSON 格式的数据：

```json
{
  "event": "monitor_triggered",
  "timestamp": "2025-01-15T10:30:00Z",
  "monitor": {
    "id": "monitor_123",
    "name": "BTC 地址监控",
    "type": "wallet"
  },
  "data": {
    // 监控相关数据
  }
}
```

## 安全配置

<Warning>
  建议启用签名验证，确保接收到的请求来自 AlertDog。您可以在URI配置时设置唯一字符。
</Warning>

使用说明：配置完成后，监控触发时 AlertDog 会向您的 Webhook URL 发送请求。您可以在自己的应用中处理这些请求，实现自定义的通知逻辑。

# 常见问题

<AccordionGroup>
<Accordion title="Webhook URL 必须是 HTTPS 吗？">
  为了安全起见，我们强制使用 HTTPS。
</Accordion>

<Accordion title="Webhook 请求超时时间是多久？">
  Webhook 请求超时时间为 5 秒。如果您的服务器响应时间过长，可能导致请求失败。
</Accordion>
</AccordionGroup>
