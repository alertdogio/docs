# AlertDog 文档 Snippets 使用指南

## 概述

本目录包含可重复使用的文档片段（Snippets），用于在多个文档中保持内容的一致性，减少维护成本。

## 目录结构

```
snippets/
├── zh/                           # 中文 Snippets
│   ├── step-sign-in.mdx         # 注册登录步骤
│   ├── step-add-notification.mdx # 添加通知渠道步骤
│   └── step-receive-alert.mdx   # 接收通知步骤
└── en/                           # 英文 Snippets
    ├── step-sign-in.mdx         # Sign In步骤
    ├── step-add-notification.mdx # Add Notification步骤
    └── step-receive-alert.mdx   # Receive Alert步骤
```

## 使用方法

在 MDX 文件中引用 Snippet：

```mdx
<Steps>
  <Snippet file="zh/step-sign-in.mdx" />
  
  <Snippet file="zh/step-add-notification.mdx" />
  
  <Step title="添加监控">
    <!-- 你的自定义内容 -->
  </Step>
  
  <Snippet file="zh/step-receive-alert.mdx" />
</Steps>
```

## 可用的 Snippets

### 中文 Snippets

#### 1. step-sign-in.mdx
**用途**: 注册登录步骤  
**使用场景**: 所有需要用户先登录的案例文档  
**引用方式**: `<Snippet file="zh/step-sign-in.mdx" />`

#### 2. step-add-notification.mdx
**用途**: 添加通知渠道步骤  
**使用场景**: 所有需要配置通知的案例文档  
**引用方式**: `<Snippet file="zh/step-add-notification.mdx" />`

#### 3. step-receive-alert.mdx
**用途**: 接收通知步骤说明  
**使用场景**: 所有监控配置完成后的说明  
**引用方式**: `<Snippet file="zh/step-receive-alert.mdx" />`

### 英文 Snippets

对应中文版本的英文翻译，路径格式为 `en/snippet-name.mdx`

## 最佳实践

### ✅ 推荐做法

1. **保持 Snippet 通用性**: Snippet 内容应该适用于所有引用场景
2. **及时更新**: 如需修改通用步骤，只需更新对应的 Snippet 文件
3. **同步中英文**: 修改中文 Snippet 时，记得同步更新英文版本
4. **添加注释**: 在复杂的 Snippet 中添加必要的注释说明

### ❌ 避免做法

1. **避免过度定制**: 如果某个步骤需要大量定制，不适合做成 Snippet
2. **避免嵌套 Snippet**: 不要在 Snippet 中引用其他 Snippet
3. **避免跨语言引用**: 中文文档只引用 zh/ 下的 Snippets

## 添加新 Snippet

如果发现新的重复内容模式，可以按以下步骤添加：

1. 在 `snippets/zh/` 或 `snippets/en/` 下创建新的 `.mdx` 文件
2. 将重复内容提取到新文件中
3. 在所有相关文档中使用 `<Snippet file="path/to/snippet.mdx" />` 引用
4. 更新此 README 文档
5. 测试验证所有引用位置显示正常

## 注意事项

- Snippet 路径是相对于 `snippets/` 目录的
- Snippet 文件必须是有效的 MDX 格式
- 修改 Snippet 后需要重新构建文档才能看到效果
- 确保 Snippet 内容在所有引用场景下都适用
